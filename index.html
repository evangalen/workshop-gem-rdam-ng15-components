1. Introductie

Tijdens deze mini-workshop willen we mensen bekend laten raken met het gebruik van components in AngularJS 1.5.

Hierbij zullen we stap voor stap een AngularJS "1.0" applicatie omzetten naar AngularJS 1.5 in combinatie met de nieuwe .component methode.

Voor de workshop heb afgezien van een moderne browser (Chrome of Firefox geadviseerd) geen aanvullende software nodig. Dit omdat we alles doen m.b.v. Plunker doen, een simple editor voor JavaScript / HTML / CSS web apps met ingebouwde (auto-)preview functionaliteit.

De applicatie die we gaan moderniseren naar AngularJS 1.5 is "Simple Reddit", een eenvoudige reddit kloon die:

  • een lijstje van links toont gesorteerd op de voting score

  • per link de mogelijkheid tot een "up"- danwel "down"-vote geeft

  • een formulier bevat met de mogelijkheid om een nieuwe link toe te voegen

2. Kennismaking met Plunker

Alvorens we echt aan de slag gaan met het refactor-en van de "Simple Reddit" web app gaan we onszelf eerst bekend maken met Plunker.

Hiertoe openen we (alvast) de Plunker van de beginsituatie van onze web app middels deze link.

Van links naar rechts zie je de volgende panels in Plunker:

  • Uiterst links staat een overzicht van o.a. de bestanden van de web app; van de bestanden is er altijd 1 geselecteerd.

  • Een panel rechts daarnaast bevat het geselecteerd bestand geopend in een editor.

  • En weer rechts waarvan vind je een preview van web app; deze preview wordt bij wijziging continu in de editor bijgewerkt.

  • Uiterste rechts staat een vertikale toolbar; standaard is het "oog" geselecteerd wat aangeeft dat de preview mode actief is. Door hierop te klikken kunnen je de preview mode deswewenst uit- en -inschakelen.

Daarnaast staan er links bovenin de pagina nog wat relevante knoppen. Hierin staat op dit moment de knoppen "Fork", "New" en "Stop". Door op "Stop" te drukken kun je net al bij het "oog" de preview mode uitschakelen en zal deze wijzigen in "Run" wat je de mogelijkheid geeft om de preview weer te herstellen.

Op dit moment is er nog geen "Save" knop aanwezig.
Verderop in de workshop zullen we een "Fork" doen, waarna deze beschikbaar zal komen.

3. Beginsituatie in AngularJS 1.0

Open de Plunker van de AngularJS 1.0 begin situatie van onze web app middels deze link.

De AngularJS "1.0" versie van "Simple Reddit" bestaat uit de volgende bestanden:

  • index.html: index HTML pagina met <script> en <link> tags voor libraries en eigen sources

  • style.css: custom CSS styles

  • app.module.js: de AngularJS (dependency injection) module declaratie

  • article.service.value.js: bevat de "Article" service, een constructor functie (vergelijkbaar aan een class in Java) met daarin de properties "title", "link" en "votes" en methodes "voteUp" en "voteDown".

  • app.html: een HTML template die ge-include vanuit "index.html" met:

    • <div ng-controller="RedditAppController"> als root element

    • een include van "add-article.html"

    • een loop (ng-repeat) over de sortedArticles() waarbij voor iedere article er een include van "article.html" wordt gedaan

  • app.controller.js: bevat de RedditAppController die het volgende op $scope plaatst:

    • articles property: een voorgevulde array met Article instanties

    • addArticle methode: voegt een nieuwe Article instantie toe aan articles o.b.v. zijn title en link argumenten.

    • sortedArtcles methode: retourneert de articles array gesorteerd op votes.

  • add-article.html + add-article.controller.js: HTML template met daarin een form voor toevoegen van artikelen en bijbehorende AngularJS controller

  • article.html + article.controller.js: HTML template met bijbehorende AngularJS controller waarin de link, host en score getoond wordt van een Artikel inclusief de mogelijkheid tot "up"- of "down"-vote

Kenmerkend voor de AngularJS 1.0 versie van deze applicatie is:

  • dat deze nog gebruikt van losse (view-loze) controllers

  • dat $scope wordt gebruikt als glue tussen de controllers en de views. ondanks dat $scope veel goeds heeft gebracht, is het ook een probleem veroorzaker voor AngularJS apps namelijk: Scope Soup.

  • sommige constructies zoals $scope.$parent. niet bepaald de schoonheidsprijs verdienen.

Klik zelf ook even snel door de diverse bestanden heen om een indruk te krijgen van onze applicatie, draai de applicatie ook even en speel ermee.

Merk op dat alle JavaScript code binnen (function() { …​ }()); staat.
Deze Immediately-Invoked Function Expression (IIFE) voorkomt dat variabelen onbedoeld globaal worden.
Daarnaast gebruiken van "strict mode" doordat we overal als eerste statement 'use strict'; gebruiken.

4. Upgrade naar versie 1.5 van AngularJS

Voordat we de upgrade naar versie 1.5 van AngularJS doen gaan we eerst een "Fork" doen van de Plunker.
Druk op "Fork" in de toolbar (naast de "New" knop); het adres zal nu wijzigen in je eigen persoonlijke unieke URL.
Zodra je nu een wijziging maakt (bijv. spatie toevoegen in de "index.html") dan zal een "Save" button verschijnen en kan je gemaakte wijzigen opslaan en op een later moment (als je de URL bewaart of in een tab open laat staan) eraan verder werken.

De eerste stap van de refactoring is dat we onze web app ook daadwerkelijk AngularJS 1.5 laten gebruiken.

Zorg ervoor dat in "Plunker" de "index.html" geopend is.
Zoek de regel met <script src="https://code.angularjs.org/1.0.8/angular.js" en wijzig hier data-require="angular.js@1.0.x" naar data-require="angular.js@1.5.x".

Door deze wijziging laten we Plunker voor ons automatisch de URL van de script tag aanpassen naar de laatste versie van 1.5.x.

Klik in de vertikale toolbar, uiterste rechts, in Plunker op het "boek" icoon (gepositioneerd onder het "oog" icoon).
De preview wordt nu vervangen door informatie over de gebruikte packages. Klik nu op de rode "1" achter "Search packages…​".
Vervolgens zie je dan dat versie van Angular 1.0.8 "out of date" is, upgrade naar 1.5.x door op de groene knop "Update to 1.5." te drukken.
De script tag wordt nu bijgewerkt en nu kunnen we daadwerkelijk ook 1.5 features gaan gebruiken.

5. 1ste component: "redditAddArticle"

De HTML template "add-article.html" en "add-article.controller.js" gaan we nu als 1ste omvormen tot een AngularJS 1.5 component.

Hiertoe passen we de naam van het .js bestand aan:

  • dubbel-klik "add-article.controller.js" en wijzig de naam in "add-article.component.js"

  • pas in de "index.html" ook de naam aan bij zijn "script" tag

Open nu "add-article.component.js" (die eerder nog "add-article.controller.js" hete).

Wijzig de regel met

.controller('RedditAddArticleController', RedditAddArticleController);

door de volgende code:

.component('redditAddArticle', {
    bindings: {
        onAddArticle: '&'
    },
    templateUrl: 'add-article.component.html',
    controller: RedditAddArticleController
});

Deze wijziging (ondanks de extra regels code) is een hele verbetering ten opzichte van eerst:

  • door een definieren van een output-binding "onAddArticle" kan nu de gebruiker van dit component zelf bepalen wat er moet gebeuren na een "submit" van de form. Hetgeen een hele verbetering is ten opzichte van de expliciete $scope.$parent.addArticle($scope.newTitle, $scope.newLink) van eerst.

  • het component bepaalt zelf zijn HTML template in plaats van dat de HTML template d.m.v. ng-controller bepaald welke controller er gebruik moet worden. In de praktijk zijn HTML template en bijbehorende (in dit geval component) controller toch altijd tight-coupled en nu is tenminste vanuit de JavaScript code duidelijke werke template er wordt gebruikt.

Nu moeten de we RedditAddArticleComponentController constructor function aanpassen:

  • verwijder het $scope argument van de function

  • voeg als eerste regel de volgende statement toe gevolgd door een wit regel:
    var $ctrl = this;

  • zoek en vervang alle voorkomens van $scope. door $ctrl. behalve bij $scope.$parent maar daarentegen wel bij de $scope.newTitle en $scope.newLink op dezelfde regel.

  • wijzig de regel
    $scope.$parent.addArticle($ctrl.newTitle, $ctrl.newLink); in: $ctrl.onAddArticle({$event: {title: $ctrl.newTitle, link: $ctrl.newLink}});

Door de voorgaande wijzigingen in de controller:

  • wordt er nu geen gebruik meer gemaakt van $scope maar plaatsen we alles op de this van de controller instantie.

  • zal in zijn HTML template de controller as $ctrl op de $scope beschikbaar gesteld worden.

  • wordt nu niet meer direct de "addArticle" methode van de parent scope aangeroepen; in plaats daarvan gebruiken we nu de output-binding onAddArticle en geven we hieraan een $event met daarin een object bestaande uit "title" en "link" mee.

We gaan nu de HTML template "add-article.html" aanpassen:

  • verwijder het attribuut ng-controller="RedditAddArticleController"

  • wijzig ng-model="newTitle" in ng-model="$ctrl.newTitle"

  • wijzig ng-model="newLink" in ng-model="$ctrl.newLink"

  • wijzig ng-click="addArticle(newTitle, newLink)" in ng-click="$ctrl.addArticle()"

Tenslotte moeten we nu ook daadwerkelijke het "redditAddArticle" component gaan gebruiken. Open "app.html" en wijzig
<div ng-include="'add-article.html'"></div>
in

<reddit-add-article
    on-add-article="$ctrl.addArticle($event.title, $event.link)">
</reddit-add-article>

Het resultaat is een geïsoleerd component dat met de buitenwereld communiceert doormiddel van de output-only (event) binding "onAddArticle".

6. 2de component: "redditArticle"

Nu we eerder het "redditAddArticle" component hebben gemaakt, gaan we nu een component maken voor de weergave van een artikel en gaan we nu met "article.html" en "article.controller.js" aan de slag.

Eerst passen we wederom de naam van het .js bestand aan:

  • dubbel-klik "article.controller.js" en wijzig de naam in "article.component.js"

  • pas in de "index.html" ook zijn naam aan bij zijn "script" tag

Open nu "article.component.js" (die eerder nog "article.controller.js" heete).

Wijzig de regel met

.controller('RedditArticleController', RedditArticleController);

door de volgende code:

.component('redditArticle', {
     bindings: {
         article: '<bindArticle',
     },
     templateUrl: 'article.component.html',

     controller: RedditArticleComponentController
});

Deze wijziging (ondanks de extra regels code) is een hele verbetering ten opzichte van eerst:

  • we gebruiken nu een one-way input-binding "article" inplaats deze te gebruiken van een inherited parent scope; om duidelijk in de HTML zichtbaar te maken dat we hier als attribuut waarde een AngularJS expressie verwachten (i.p.v. een fixed string waarde) gebruiken we i.p.v. article de attribuut naam bind-article. Voor meer info over het waarom zie mij blog post hierover.

  • de component bepaald zelf zijn HTML template in plaats van dat de HTML template bepaald d.m.v. ng-controller welke controller er gebruik moet worden.

Nu moeten de we RedditAddArticleController constructor function aanpassen:

  • verwijder het $scope argument van de function

merk op dat RedditAddArticleController constructor helemaal leeg is.
Voor de werking van .component dient namelijk altijd een controller gespecificeerd te worden.
Verder acteert het als een placeholder voor de optionele oefeningen even verder op.

We moeten de HTML template "article.html" aanpassen:

  • verwijder de attribuut ng-controller="RedditArticleController"

  • wijzig {{ article.votes }} in {{ $ctrl.article.votes }}

  • wijzig href="{{ article.link }}" in href="{{ $ctrl.article.link }}"

  • wijzig {{ article.title }} in {{ $ctrl.article.title }}

  • wijzig ({{ article.domain() }}) in ({{ $ctrl.article.domain() }})

  • wijzig ng-click="article.voteUp()" in ng-click="$ctrl.article.voteUp()"

  • wijzig ng-click="article.voteDown()" in ng-click="$ctrl.article.voteDown()"

Last but not least moeten we nu ook daadwerkelijke de "redditArticle" component gaan gebruiken. Open "app.html" en wijzig
<div ng-include="'article.html'"></div>
in

<reddit-article ng-repeat="article in sortedArticles()"
        bind-article="article">
</reddit-article>

We hebben nu eerste versie van het "redditArticle" component. Voor de (one-way) input-only "article" binding hebben we nu de requirement op "article" expliciet gemaakt i.p.v. dat we deze gebruiken van de inherited parent scope.

Desondanks is er ruimte voor verbetering die aan jullie zelf laat als oefening:

  • in plaats van een Article instantie meer te geven, zouden we er ook voor kunnen kiezen om de "title", "link" en "votes" als losse input-only bindings mee te geven.

  • door output-only "onVoteUp" en "onVoteDown" bindings zouden we ook het aanroepen van "voteUp" en "voteDown" op Article buiten de "redditArticle" component kunnen plaatsen; in de aanroepende HTML zou dan een methode van de RedditAppController (of beter nog van de "redditApp" component) kunnen worden aangeroepen die vervolgens de daadwerkelijke "voteUp" en "voteDown" doet op Artikel.

  • in Article staat de "domain" methode in de HTML template van "redditArticle" component wordt gebruik om tussen haakjes (in grijze tekst) de host name van de link te tonen. op dit moment wordt de "domain" methode enkel gebruikt door de "redditArticle" component.
    Misschien is het wel beter als we deze (presentatie) logica verplaatsten naar binnen het component?

7. Conclusie

Na hopelijk (ook gezien de korte tijd) een 2 tal components te hebben gemaakt in AngularJS 1.5 hoop ik dat ik jullie enthousiast heb kunnen maken voor components.

Mocht niet alles gelukt zijn dan vind je hier de oplossing van de workshop.

Mocht je geinteresseerd zijn? Eigenlijk in de migratie naar componenten nog niet helemaal af ;-) De "app.controller.js" en bijbehorende "app.html" zo nog kunnen worden omgevormd naar een "redditApp" component. Dit is een zogenaamd "app"-component zonder enige bindings.

De migratie van AngularJS applicaties naar components is belangrijk aangezien components de toekomst van Angular zijn. Zo zijn in Angular 2 de ng-include en ng-controller attribute directives niet meer aanwezig, en dien je wel components te gebruiken.
Daarnaast biedt het gebruik van components in AngularJS 1.5 een migratie pad naar Angular 2. Doormiddel van de "ngUpgrade" module is het namelijk mogelijk om een bestaand AngularJS 1.5 .component te gebruiken binnen een Angular 2 applicatie en kun je dus (tijdelijk) 1 hybride web app maken met daarin zowel AngularJS 1.5 en Angular 2.
